ARIA Landmark Roles

The Accessible Rich Internet Applications (WAI-ARIA) defines a set of roles known as document landmarks. These roles break the page into easy to navigate sections. Landmarks assist users of assistive technologies to navigate the different sections of a page, the flow of the page and its overall structure more easily by describing the type or purpose of content. Landmarks are added to a page by specifying the role attribute on certain HTML elements.

You can add ARIA landmarks to define the following regions:

Banner role:

The banner role is applied to an area of the page containing site-oriented information. It denotes the portion of a page where a company logo, site slogan, the main heading for the page or the like would be found. Typically this appears at the top of the page spanning the full width. It's important that you can use the banner role only once within any HTML page.

Navigation role:

The navigation role is used to identify areas of the page containing collections of navigational links. It can be used multiple times on the HTML page, so an area with a navigation landmark might contain links for navigating within the page, or for navigating elsewhere on the site.

Main role:

The main role is used to identify the primary content area of the page. It can't be used more than once on any HTML page.

Complementary role:

The complimentary role is applied to an area containing content that is related to, but separate from, the main content topic. For example, Sports News and Business News on a typical news page. In other words, the area holding related links and associated content. It should be meaningful when separated from the main content.

Contentinfo role:

The contentinfo role is applied to an area containing page specific information. It denotes the section of a page that contains the copyright notice, link to privacy statement etc. It's typically used for the footer of a site.

Search role:

The search role is pretty self-explanatory, it denotes the section of a page that starts your search feature.

Form role:

The form role is used to identify areas of a page containing form fields. It can be used multiple times on a page, and for almost any kind of web form.

Application role:

The application role is applied to areas of the page that contain application style content and functionality. It denotes a section of a page that provides features and a very rich experience with widgets not found in standard HTML. It tells the assistive technologies to switch from normal browsing mode to application browsing mode. It's perhaps the most significant of the landmark roles because itâ€™s the only one that forces the screen reader to behave differently.

Example:

Here's the basic HTML code:

<div id="header">Company Name</div>
<nav>
  <ul>
  <li><a href="#">Home</a></li>
  <li><a href="#">About us</a></li>
  <li><a href="#">Contact us</a></li>
  </ul>
</nav>
<div>
  <p>Main content will come here!</p>
</div>

HTML code after applying ARIA roles:

<div id="header" role="banner">Company Name</div>
<nav role="navigation">
  <ul>
  <li><a href="#">Home</a></li>
  <li><a href="#">About us</a></li>
  <li><a href="#">Contact us</a></li>
  </ul>
</nav>
<div role="main">
  <p>Main content will come here!</p>
</div>

As you can see in the above example, adding role attribute to a container element gives it the most relevant value and allows the assistive technology to provide a consistent navigation experience to its users.
